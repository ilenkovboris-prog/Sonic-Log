# Sonic Log Prediction: BiLSTM for Cross-Well DT Estimation

## Восстановление акустического каротажа (DT) по стандартным каротажным кривым

[![Python](https://img.shields.io/badge/Python-3.8+-blue.svg)](https://python.org)
[![PyTorch](https://img.shields.io/badge/PyTorch-2.0+-red.svg)](https://pytorch.org)

### Задача

Акустический каротаж (DT / Sonic log) — дорогостоящее измерение, которое
записывается не на всех скважинах. Из 5 скважин месторождения Volve
только 3 имеют DT. Этот проект восстанавливает отсутствующий DT по стандартным
каротажным кривым (GR, RHOB, NPHI, PEF) с использованием Bidirectional LSTM.

### Результаты

Тестирование на контрольной скважине F-11A (модель НЕ видела при обучении):

| Модель | MAE (мкс/фт) | R² |
|---|---|---|
| Бейзлайн (среднее) | 12.76 | — |
| Random Forest | 3.41 | 0.893 |
| BiLSTM v1 (окно 21) | 3.31 | 0.904 |
| BiLSTM v2 (окно 11 + градиенты) | 3.21 | 0.895 |
| **Ensemble (RF + BiLSTM)** | **3.12** | **0.904** |

Ошибка 3.12 мкс/фт при диапазоне DT 53–127 мкс/фт — это ~4% от размаха.

### Почему это работает

В отличие от виртуального расходомера (см. volve-virtual-flowmeter), где
cross-well перенос не работал, здесь cross-well предсказание успешно потому что:

- Каротажные данные определяются геологией, а не действиями оператора
- Связь GR/RHOB/NPHI → DT одинакова для всех скважин одного месторождения
- Данных ~25 000 точек для обучения (vs ~2 000 дней добычи)

### Ключевые решения

**Bidirectional LSTM.** Каротаж — это последовательность по глубине, не по времени.
Контекст сверху и снизу одинаково важен, поэтому используется двунаправленная
архитектура. Выход берётся из центра окна, а не из последней точки.

**Градиентные фичи.** Добавлены производные каротажных кривых (dGR/dz, dRHOB/dz и т.д.),
которые явно кодируют границы пластов — резкие изменения свойств на контактах.

**Ensemble RF + LSTM.** Random Forest точнее ловит амплитуду скачков (каждая точка
независимо), LSTM лучше видит контекст и переходные зоны. Оптимальная комбинация
побила оба отдельных метода.

### Архитектура

```
Input: (batch, 11, 8)  — 11 точек × 8 фичей (4 кривые + 4 градиента)
  │
  ▼
Bidirectional LSTM (hidden=64)  — читает разрез вверх и вниз
  │
  ▼
Dropout(0.2)
  │
  ▼
Linear(128 → 1)  — предсказание DT в центре окна
  │
  ▼
Output: DT (мкс/фт)

+ Random Forest (100 деревьев) для ensemble
```

### Входные фичи

| Кривая | Описание | Что измеряет |
|---|---|---|
| GR | Gamma Ray | Радиоактивность породы → литология |
| RHOB | Bulk Density | Плотность → пористость |
| NPHI | Neutron Porosity | Водородный индекс → пористость |
| PEF | Photoelectric Factor | Фотоэффект → минералогия |
| + градиенты | dGR/dz, dRHOB/dz... | Скорость изменения → границы пластов |

### Выводы

1. **Cross-well предсказание каротажа работает** — в отличие от добычных данных
2. **Ensemble RF + LSTM побеждает** оба метода по отдельности
3. **BiLSTM лучше обычного LSTM** для каротажа (контекст симметричен)
4. **Градиенты фичей помогают** на границах пластов
5. **Ошибка ~4%** от диапазона DT — достаточно для практического применения
